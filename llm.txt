# AIMS RDR — LLM Context Pack (Nacala Coal Patio Volume Detection)

Last updated: 2026-07-15
Purpose: Give future agents enough ground truth to extend the system without rediscovering failures.

================================================================================
1. PRODUCT INTENT
================================================================================

Site: Nacala Port & Coal Field (Mozambique) — coal stockyard, NOT roads.
Domain language:
  - patio = parallel storage lane (A / B / C)
  - pile / stockpile = named feature NC_CYA_P_01 … NC_CYC_P_*
  - lining material = flat pad (low slope), not a mound
  - volume detection = loose coal volumes + change over survey dates
  - Do NOT call features potholes/ruts/heave for this site

Primary deliverable: MatrixGeo-style PDF
  - Measurement Chainage & Limits overview
  - CALCULATION OF LOOSE COAL VOLUMES PATIO - A/B/C (table + DEM strip)
  - L-SECTION STOCKPILE PROFILES (elevation curve + shaded plan)

UI: Next.js monitoring dashboard at /monitor/*
  - /monitor/parameters = Patio Volumes (Generate PDF)

4. CRS RULE (MOST IMPORTANT BUG CLASS)
================================================================================

Pile polygons:  EPSG:32737  (WGS 84 / UTM zone 37S)
DEM products:   EPSG:21037  (Arc 1960 / UTM zone 37S)

If you sample DEM with polygon XY without warping:
  - elevation returns nodata (-32767)
  - L-section graphs are EMPTY grids
  - plan views are flat grey with only red outlines
  - patio strips look wrong / similar

Fix location: patio_report_figures.py
  _PILE_CRS = "EPSG:32737"
  warp rings with rasterio.warp.transform before from_bounds / sampling

Verification: after warp, NC_CYA_P_02 centroid samples ~16–17 m elev on Feb DEM.

================================================================================
5. REPORT METRICS
================================================================================

Table columns (MatrixGeo):
  Name | Pile Name | Date of Survey | Net Volume (m³) | Enclosed Area (ha)
  | Chainage | Product | Maximum Height of the Pile | Avg. Angle of Repose

Derived:
  max_height_m = max_elev_m - min_elev_m
  avg_angle_repose = properties.AVG_SLOPE_ (shown only for stockpiles)
  lining if AVG_SLOPE_ < 8° (or near-zero volume pad) → Product "Lining Material",
    blank height/repose
  stockpile product grades by height tier → VT4T / MABU / MLVC / Anastasia labels

Chainage:
  project pile onto patio long axis; 0 at P_01 end; MatrixGeo L-section overrides
  for known piles (e.g. NC_CYA_P_02 = 360-430)

Volumes (current):
  stage-3.json net volumes
  Patio A ≈ 60,783 m³ (sample Final Sheet was 60,843 — close)
  Patio B ≈ 114,889 m³ (sample Final Sheet 67,926 — different sheet; restore XLSX to match)
